## AS-IS（現状）

| 課題 | 影響 |
|---|---|
| 商品ラベル・食品表示 PDF の情報が手入力または散在している | 転記ミス・検索困難 |
| 名刺 PDF 版 RPA はあるが商品向け列・プロンプトではない | 流用にはスキーマ・重複キーの再設計が必要 |

## TO-BE（目指す姿）

Google ドライブの指定フォルダに置いた商品 PDF を GAS + Gemini で自動読み取りし、商品情報を構造化 JSON に変換して Google スプレッドシート「商品DB」へ追記する。正常処理後は規則に従い PDF をリネームし処理済みフォルダへ移動し、失敗 PDF はエラーフォルダへ隔離する。

## 機能要件（FR）

| FR コード | 機能名 | 優先度 (MoSCoW) | 概要 |
|---|---|---|---|
| FR-PDF-001 | PDF 自動検知 | Must | 指定 Drive フォルダ内の未処理 PDF を列挙する |
| FR-PDF-002 | 商品情報抽出 | Must | Gemini で商品 PDF から商品情報 JSON を抽出する |
| FR-PDF-003 | 処理済み移動 | Must | 正常処理後、PDF を処理済みフォルダへ移動する |
| FR-PDF-004 | PDF ファイル名自動変更 | Should | 抽出成功後、規則に従い Drive 上の PDF 名を変更する |
| FR-SHT-001 | DB 書き込み | Must | 抽出 JSON を商品DB シートへ 1 行追加する |
| FR-SHT-002 | 重複チェック | Should | JAN / 商品コード / 複合キーで既存行を検知する |
| FR-NTF-001 | エラー通知 | Must | Slack へ異常を通知する |
| FR-NTF-002 | ファイル隔離 | Must | 解析失敗 PDF をエラーフォルダへ移動する |

## 非機能要件（NFR）

| NFR コード | 種別 | 優先度 | 概要 |
|---|---|---|---|
| NFR-SEC-001 | セキュリティ | Must | Gemini API キー、Slack Webhook、Drive フォルダ ID をコードやログに出さない |
| NFR-OPS-002 | 運用・保守 | Must | 429/500 系リトライ、401 即停止・バッチ中断 |
| NFR-PERF-001 | パフォーマンス | Should | Gemini 無料枠内での日次処理件数を想定 |

## MoSCoW サマリ

| 区分 | 件数 |
|---|---|
| Must | PDF 検知・抽出・Sheets 追記・フォルダ移動・エラー隔離・秘密情報管理 |
| Should | 重複チェック、期限正規化、栄養成分の単位保持、PDF ファイル名自動変更 |
| Could | 複数商品対応、栄養成分別シート、信頼度メモ |
| Wont | 在庫管理、人手確認ワークフロー、OCR 画像補正（初期版） |

## 採番ルール

### コード体系

```
[TYPE]-[CATEGORY]-[NNN]

TYPE:  FR = 機能要件  /  NFR = 非機能要件

CATEGORY（FR 用）:
  PDF   = PDF 取込・解析・ファイル移動
  SHT   = スプレッドシート連携
  NTF   = 通知・エラー処理

CATEGORY（NFR 用）:
  SEC   = セキュリティ
  PERF  = パフォーマンス
  OPS   = 運用・保守

NNN: カテゴリ内の連番（001〜）。欠番は振り直さない。
```

## 機能要件（FR）

### PDF 取込・解析（PDF）

| FR コード | 機能名 | 実装状況 | 概要 | 備考 |
|---|---|---|---|---|
| FR-PDF-001 | PDF 自動検知 | 未着手 | インプットフォルダ内の未処理 PDF を検出・一括処理 | Must |
| FR-PDF-002 | 商品情報抽出 | 実装済 | Gemini で PDF → 商品 JSON | Must。PoC: 手動 1 件 |
| FR-PDF-003 | 処理済み PDF 移動 | 未着手 | 正常処理後、02_処理済みフォルダへ移動 | Must |

### スプレッドシート連携（SHT）

| FR コード | 機能名 | 実装状況 | 概要 | 備考 |
|---|---|---|---|---|
| FR-SHT-001 | DB 書き込み | 実装済 | 抽出 JSON を商品DB 最終行へ追加 | Must。PoC: 重複チェックなし |
| FR-SHT-002 | 重複チェック | 未着手 | JAN / 商品コード / 複合キー | Should |

### 通知・エラー処理（NTF）

| FR コード | 機能名 | 実装状況 | 概要 | 備考 |
|---|---|---|---|---|
| FR-NTF-001 | エラー通知 | 未着手 | Slack へ API/システム例外を通知 | Must |
| FR-NTF-002 | ファイル隔離 | 未着手 | 解析失敗 PDF を 03_エラーフォルダへ移動 | Must |

## 非機能要件（NFR）

| NFR コード | 種別 | 機能名 | 実装状況 | 概要 |
|---|---|---|---|---|
| NFR-SEC-001 | セキュリティ | 秘密情報管理 | 未着手 | Script Properties、Git 除外 |
| NFR-OPS-002 | 運用・保守 | リトライ・停止 | 未着手 | 429/500 系リトライ、401 は即停止 |

## 実装状況の凡例

| 値 | 意味 |

| 未着手 | 実装していない |
| 実装済 | 実装・動作確認済み |
| 廃止 | スコープ外にした |
| 検討中 | 要件が未確定 |

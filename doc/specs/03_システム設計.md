## アーキテクチャ概要

### 処理フロー

```
[商品 PDF 投入]
      ↓
[01_インプットフォルダ]  ← Google ドライブ
      ↓ 定時トリガー / 手動実行
[main.js] ──→ [config.js]     設定読込
      │
      ├─→ [gemini.js]          PDF → 商品 JSON（ジャンルはマスタから 1 件選択）
      ├─→ [spreadsheet.js]     JSON → 商品DB 行追加・重複チェック
      ├─→ [drive.js]           ファイル名生成・リネーム（FR-PDF-004）
      ├─→ 正常時: [02_処理済みフォルダ] へ PDF 移動
      └─→ 異常時: [notification.js] → Slack
                解析失敗時: [03_エラーフォルダ] へ PDF 隔離
```

### モジュール責務

| モジュール | 責務 |
|---|---|
| `main.js` | PDF 列挙、1 件処理、バッチ継続、エラー分類 |
| `drive.js` | 入力・処理済み・エラーフォルダの 3 分離、PDF リネーム（`buildProductPdfFileName_` / `renamePdfFileSafely_`） |
| `genreList.js` | ジャンルマスタ一覧・正規化 |
| `gemini.js` | PDF を inlineData で渡す、JSON 応答、リトライ |
| `spreadsheet.js` | schema 列順で行組み立て、商品向け重複チェック |
| `schema.js` | 列定義・Gemini キー・正規化（正本） |
| `setupSpreadsheet.js` | 商品DB シート作成・ヘッダー再適用 |
| `ui.js` / `ControlPanel.html` | メニュー・サイドバー |
| `notification.js` | Slack 通知 |

## データモデル

> **正本**: `gas/src/schema.js` / [product-schema-design.md](../reference/setup/product-schema-design.md)

### スプレッドシート列

シート名: `商品DB`（開いているスプレッドシート内の新規タブ）

主要列（左から）: 商品名、ジャンル、期限種別、期限日、アレルゲン、原材料、内容量、栄養成分（基準量 + 主要 5 項目）、メーカー・補足メモ、右端に JAN・商品コード・処理日時・元ファイル・Drive ファイル名。全 25 列。詳細は `schema.js` を参照。

### Gemini 出力 JSON スキーマ

```json
{
  "product_name": "",
  "genre": "",
  "maker": "",
  "brand": "",
  "jan_code": "",
  "product_code": "",
  "expiration_type": "",
  "expiration_date": "",
  "net_content": "",
  "ingredients": "",
  "allergens": "",
  "storage_method": "",
  "country_of_origin": "",
  "serving_size": "",
  "energy_kcal": "",
  "protein_g": "",
  "fat_g": "",
  "carbohydrate_g": "",
  "salt_equivalent_g": "",
  "nutrition_notes": "",
  "confidence_notes": "",
  "notes": ""
}
```

GAS 書き込み時は上記 JSON に加え、システム列 `processed_at`・`source_file`・`drive_file_name` を付与する。

### PDF ファイル名規則（FR-PDF-004）

正常処理後、処理済みフォルダへ移動する直前に Drive 上の PDF 名を変更する。

| 項目 | 規則 |
|---|---|
| パターン | `{yyyymmdd}_{genre}_{product_name}.pdf` |
| 日付プレフィックス | `expiration_date` が正規化可能ならその日付（YY.MM 刻印は当月末日）。それ以外は `processed_at` の Asia/Tokyo 日付 |
| ジャンル | `genreList.js` のマスタから Gemini が 1 件選択。一覧外は `その他` |
| サニタイズ | Drive 禁止文字 `/ \ : * ? " < > \|` を `_` に置換。連続空白は `_` に統一 |
| 最大長 | 拡張子除き 200 文字以内に切り詰め |
| 衝突回避 | 同一フォルダに同名がある場合 `_2`、`_3` … の連番サフィックスを付与 |
| 監査 | `source_file` はスキャン時の元名を保持。リネーム後の名前は `drive_file_name` に記録 |
| 失敗時 | リネーム失敗は処理全体を失敗扱いにしない。元名のまま移動しログ出力 |

### 重複チェック（FR-SHT-002）

優先順位は JAN → 商品コード → 複合キー（`product_name + maker`）。**全経路で `expiration_date` を比較に用いる。**

| 優先度 | ベースキー | ベースキーの正規化 |
|:---:|---|---|
| 1 | `jan_code` | 数字以外を除去 |
| 2 | `product_code` | 空白除去 + 大文字化 |
| 3 | `product_name + maker` | trim + 空白正規化 |

期限日（`expiration_date`）は `normalizeExpirationDateToIso_` で `YYYY-MM-DD` に正規化して比較する。

| 条件 | 判定 |
|---|---|
| ベースキー不一致 | 非重複（別行として追記） |
| ベースキー一致・両方に期限あり・日付が異なる | 非重複（別行として追記） |
| ベースキー一致・両方に期限あり・日付が同じ | 重複（追記スキップ） |
| ベースキー一致・どちらか一方（または両方）に期限なし | 重複（期限不問として追記スキップ） |

## セキュリティ・運用（設計レベル）

- 秘密情報は Script Properties（GAS）および `config/.env`（ローカル、Git 除外）
- 列変更は **必ず `schema.js` 経由**。スプレッドシート UI だけで列を移動しない
- 技術選定: [doc/adr/0003-gas-runtime.md](../adr/0003-gas-runtime.md)

### 入力運用（1 PDF 1 商品）

複数商品を 1 PDF に含めない。これはコードで複数行へ展開するのではなく、**運用ルール**で 1 PDF = 1 シート行の形を守る。

| 観点 | 方針 |
|---|---|
| 正 | 商品ごとに PDF を分けてインプットフォルダへ配置する |
| 非 | 1 PDF から複数商品を抽出して複数行追記する機能は実装しない |
| 安全網 | 逸脱 PDF が投入された場合、Gemini は主対象 1 件のみ返し `confidence_notes` に記録する |

他ツール（名刺版 RPA 等）と同じ「1 ファイル 1 レコード」前提を維持し、連携・応用時の齟齬を避ける。

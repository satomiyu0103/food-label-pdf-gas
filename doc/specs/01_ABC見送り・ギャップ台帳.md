# ABC見送り・ギャップ台帳

最終更新: 2026-07-01

PoC や学習期間で「意図的に切った点（A）」「気づきはあるが未実施（B）」「盲点（C）」を **1か所に集約** する正本です。
他ドキュメントからは **リンクのみ** で参照し、理由の二重記載を避けてください。

---

## 分類定義

| 分類 | 意味 | 正本の置き場 |
|---|---|---|
| **A** | スコープで **意図的に切った** 機能 | 詳細は [04_機能一覧.md](04_機能一覧.md) の「廃止」または MoSCoW Wont。本ファイルには GAP-ID + 理由1行のみ |
| **B** | **気づきはある** が、未実装・未検証 | 本ファイル B セクション |
| **C** | **盲点** だった／方針・時間で手を付けられなかった | 本ファイル C セクション |

### 状態の凡例

| 状態 | 意味 |
|---|---|
| 未着手 | これから取り組む |
| 対応中 | 実装またはドキュメント化中 |
| 完了 | 対応済み |
| 見送り確定 | スコープ外としてクローズ |

## いつこのファイルを更新するか

1. **機能の見送り（A）** → [04_機能一覧.md](04_機能一覧.md) の実装状況を更新。本ファイル A 索引に GAP-ID を1行追加。

## 2. フォルダ構成の意味

| フォルダ | 役割 |
|---|---|
| `src/` | 実際に動くプログラム本体 |
| `tests/` | 動作確認コード（壊れていないかを確かめる） |
| `doc/` | 仕様・設計・開発ルールの説明書 |
| `config/` | ローカル設定（`.env` など、Git 除外） |
| `data/` | 実行時に生成されるデータ（Git 除外） |
| `logs/` | 実行ログ出力先（Git 除外） |

## 3. なぜ `src/` を分けるのか

`src/` を分ける理由は、テスト時に「たまたま動いた」を防ぐためです。
コードの読み込み先を明確にすることで、開発環境と実行環境の差による事故を減らせます。
`uv sync` 後にパッケージとしてインストールされるため、`sys.path` 操作が不要になります。

## 4. レイヤの考え方（役割分担）

<!-- プロジェクトのレイヤ構成に合わせて書き直す -->

| レイヤ | 責務 |
|---|---|
| `domain/` | ビジネスロジック。外部依存なし |
| `clients/` | 外部 API への接続処理 |
| `infra/` | 設定読み込み・ログ・状態保存などの土台 |
| `sync/` / `service/` | 各部品をつないで処理を実行する司令塔 |
| `cli/` / `api/` | コマンド or リクエストを受け取り実行開始 |

# Markdown 記法マニュアル

> 最終更新: {{LAST_UPDATED}}  
> このリポジトリの `doc/` や PR 説明で使う **CommonMark + GFM（GitHub Flavored Markdown）** の早見表です。  
> Cursor / VS Code のプレビュー: `Ctrl+Shift+V`（Mac: `Cmd+Shift+V`）

各項目は **記法**（そのまま書く文字列）と **結果**（プレビューでどう見えるか）を並べています。

---

## 見出し


| レベル | 記法          | 結果のイメージ     |
| --- | ----------- | ----------- |
| 1   | `# 見出し1`    | いちばん大きいタイトル |
| 2   | `## 見出し2`   | 章タイトル       |
| 3   | `### 見出し3`  | 節タイトル       |
| 4   | `#### 見出し4` | 小見出し        |


**記法**

```markdown
## 章のタイトル
### 節のタイトル
```

**結果**

## 章のタイトル

### 節のタイトル

> `#` の直後にスペースが必要です。`#見出し` は見出しになりません。

---

## 段落・改行


| 記法                | 結果             |
| ----------------- | -------------- |
| 空行で区切る            | 別の段落になる        |
| 行末にスペース2つ + Enter | 同じ段落内で改行（狭い改行） |
| `<br>`            | 同じ段落内で改行（HTML） |


**記法**

```markdown
1行目です。
2行目です。（上に空行があるので別段落）

1行目です。  
2行目へ続く。（行末スペース2つ）
```

**結果**

1行目です。

2行目です。（上に空行があるので別段落）

1行目です。  
2行目へ続く。（行末スペース2つ）

## 強調


| 記法                    | 結果         |
| --------------------- | ---------- |
| `**太字`** または `__太字__` | **太字**     |
| `*斜体`* または `_斜体_`     | *斜体*       |
| `***太字斜体`***          | ***太字斜体*** |
| `~~取り消し~~`            | ~~取り消し~~   |


**記法**

```markdown
**重要** と *補足*、~~削除予定~~
```

**結果**

**重要** と *補足*、~~削除予定~~

## リスト

### 箇条書き（順不同）

**記法**

```markdown
- りんご
- みかん
  - 温州みかん
* バナナ（`-` と同じ）
```

**結果**

- りんご
- みかん
  - 温州みかん
- バナナ（`-` と同じ）

### 番号付き（順序あり）

**記法**

```markdown
1. 設計
2. 実装
3. テスト
```

**結果**

1. 設計
2. 実装
3. テスト

### チェックリスト（GFM）

**記法**

```markdown
- [x] 完了した作業
- [ ] 未完了の作業
```

**結果**

- 完了した作業
- 未完了の作業

## リンク・画像


| 種類     | 記法                      | 結果                                         |
| ------ | ----------------------- | ------------------------------------------ |
| リンク    | `[表示テキスト](URL)`         | [表示テキスト](https://example.com)              |
| 同じページ内 | `[見出しへ](#見出し)`          | このファイル内の見出しへジャンプ                           |
| 画像     | `![代替テキスト](画像URL)`      | 画像が表示される                                   |
| 自動リンク  | `<https://example.com>` | [https://example.com](https://example.com) |


**記法**

```markdown
[プロジェクト README](../../README.md)
![サンプル](https://via.placeholder.com/120x40?text=sample)
```

**結果**

[プロジェクト README](../../README.md)

サンプル

> 相対パスは Markdown ファイルからの位置で解決されます。

## 引用（blockquote）

**記法**

```markdown
> 他人の言葉や仕様の抜粋を引用する。
> 複数行は `>` を続ける。
```

**結果**

> 他人の言葉や仕様の抜粋を引用する。
> 複数行は `>` を続ける。

## コード


| 種類    | 記法          | 結果                           |
| ----- | ----------- | ---------------------------- |
| インライン | `コード`       | `print("hello")` のように等幅・背景付き |
| ブロック  | ````言語` で囲む | シンタックスハイライト付きブロック            |


**記法**

```markdown
`FR-AUTH-001` を実装する。

```python
def greet(name: str) -> str:
    return f"Hello, {name}"
```

```

**結果**

`FR-AUTH-001` を実装する。

```python
def greet(name: str) -> str:
    return f"Hello, {name}"
```

> このリポジトリでは `.vscode/settings.json` で Markdown の保存時自動整形を **オフ** にしています。コードフェンスやリストが崩れにくくするためです。

## 表（table）

**記法**

```markdown
| 列A | 列B |
|:---|:---|
| 左寄せ | テキスト |
| `コード` | **太字** |
```

**結果**


| 列A    | 列B     |
| ----- | ------ |
| 左寄せ   | テキスト   |
| `コード` | **太字** |



| 記法      | 意味         |
| ------- | ---------- |
| `:---`  | 左寄せ（デフォルト） |
| `:---:` | 中央寄せ       |
| `---:`  | 右寄せ        |

## エスケープ（記号をそのまま表示）


| 記法   | 結果          |
| ---- | ----------- |
| `\`* | *（斜体にならない）  |
| `\#` | #（見出しにならない） |
| `    | `           |


**記法**

```markdown
FR コードは `FR-PROJ-001` のようにバッククォートで囲むと安全です。
```

**結果**

FR コードは `FR-PROJ-001` のようにバッククォートで囲むと安全です。

## よく使う組み合わせ（doc 向け）

### 要件 ID を書く

**記法**

```markdown
- `FR-AUTH-001` ログイン: Must / 未着手
```

**結果**

- `FR-AUTH-001` ログイン: Must / 未着手

### 注意・補足

**記法**

```markdown
> **注意**: `.env` の中身はログや PR に貼らない。
```

**結果**

> **注意**: `.env` の中身はログや PR に貼らない。

### 折りたたみ（GFM / HTML）

**記法**

```markdown
<details>
<summary>クリックで開く</summary>

中身の説明
</details>
```

**結果**

クリックで開く

中身の説明

## このリポジトリで避けたい例


| 避ける                | 理由                     |
| ------------------ | ---------------------- |
| コードブロック内に秘密情報      | `.env` や API キーを貼らない   |
| 見出しの `#` のあとスペースなし | 見出しとして認識されない           |
| 表の列数が行ごとに違う        | プレビューが崩れる              |
| タブ文字でのインデント        | リストやコードが意図とずれる（スペース推奨） |

# Phase 1 PoC 実装

- 日付: 2026-06-28
- スコープ: FR-PDF-002, FR-SHT-001

## 背景・課題

ROADMAP Phase 1（PoC）として、名刺 PDF 版 `meishi-pdf-to-sheets` から商品スキーマ向けに GAS モジュールを移植する必要があった。`schema.js` と雛形のみの状態から、手動 1 件（PDF → Gemini → 商品DB 追記）まで動かすことがゴール。

## 意思決定

| 案 | 採否 |
|---|---|
| 名刺版をそのままコピーし商品向けに差し替え | 採用 |
| Phase 1 で重複チェック・バッチ・Drive 移動も実装 | 非採用（Phase 2 へ） |
| メニューから未実装の一括処理を削除 | 採用（ダミー追記メニューを追加） |
| Gemini リトライ層も Phase 1 で移植 | 採用（名刺版と同コスト） |

## 実装サマリ

- `setupSpreadsheet.js`: 商品DB タブ作成・ヘッダー 23 列・`SPREADSHEET_ID` 保存
- `spreadsheet.js`: `appendProductRecord_`・`debugAppendDummyRecord`（重複チェックなし）
- `gemini.js`: 商品向けプロンプト・API 通信・リトライ・`pocExtractFromPdfByFileId`
- `main.js`: `processPdfByFileId` 単件 E2E（PDF 移動なし）
- `ui.js`: ヘッダー再適用・ダミー追記メニュー接続

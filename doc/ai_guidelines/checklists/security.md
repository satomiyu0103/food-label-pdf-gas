# チェックリスト — セキュリティ

最終更新: 2026-05-30

> 原則の説明はガイド側に置かない。本ファイルは **セットアップ・プロトタイプ後監査** 用。  
> テンプレート適用後は `config/.env`（Git 除外）・`{{SECRET_VAR_NAME}}` をプロジェクト名に置換して使う。

各項目の形式: **観察内容** — 根拠（設計・規約へのリンク）

---

## インフラ・クラウド

- [ ] ストレージ（S3 等）が外部から誰でも見られない設定か（バケットが「公開」になっていない）  
  - 根拠: [03_システム設計.md](../../specs/03_システム設計.md)
- [ ] サーバーのネットワークが不用意に全開放になっていない（例: セキュリティグループ `0.0.0.0/0`）  
  - 根拠: [03_システム設計.md](../../specs/03_システム設計.md)
- [ ] テスト用・デモ用の公開モードが本番で有効になっていない（DEBUG 等）  
  - 根拠: [実装規約.md §7](../実装規約.md#7-禁止事項)

## シークレット（機密情報）

- [ ] API キー・秘密鍵がソース・ログに直書きされていない  
  - 根拠: [実装規約.md §7](../実装規約.md#7-禁止事項)・[03_システム設計.md](../../specs/03_システム設計.md)
  - ダメな例: `API_KEY = "sk-..."` をコードに記載
- [ ] 開発用と本番用で鍵を分けている  
  - 根拠: [03_システム設計.md](../../specs/03_システム設計.md)
- [ ] 秘密情報は環境変数（`config/.env`、Git 除外）から読み込んでいる  
  - 根拠: [03_システム設計.md](../../specs/03_システム設計.md)
- [ ] 鍵漏洩時の無効化・交換手順がある  
  - 根拠: [06_ROADMAP.md](../../specs/06_ROADMAP.md) または運用メモ

## 認証・権限

- [ ] URL の ID を変えて他人のデータにアクセスできない（IDOR 対策）  
  - 根拠: [02_要件定義.md](../../specs/02_要件定義.md)
- [ ] ユーザーごとに適切なロール・権限分けがある  

## 入力値検証

- [ ] フォーム・API の入力をサーバー側で検査している  
- [ ] ファイルアップロードの種類・サイズを制限している  

## アプリ設定

- [ ] デバッグ表示が本番で無効（DEBUG OFF）  
- [ ] `ALLOWED_HOSTS` / CORS が本番ドメインに限定されている  
- [ ] HTTPS 強制・セキュアクッキー・HSTS 等が設定されている  

  - 根拠: [02_要件定義.md](../../specs/02_要件定義.md)・[03_システム設計.md](../../specs/03_システム設計.md)

## データベース・データ

- [ ] ローカル DB ファイルがコミットされていない  
  - 根拠: [実装規約.md §5](../実装規約.md#5-セキュリティ規約)
- [ ] 本番 DB の選定・バックアップ/リストア手順が文書化されている  

## 初期アカウント・フロントエンド

- [ ] 初期アカウントに単純な固定パスワードを使っていない  
- [ ] CSRF トークン等の不正送信対策がある（Web アプリの場合）  
- [ ] 静的ファイルが本番用に集約・配信される設定  

## 運用・監視（セキュリティ観点）

- [ ] ログ・通知に秘密情報・個人情報全文が含まれない  
  - 根拠: [安全運用ガイド.md §2.3](../安全運用ガイド.md#23-ログに書かないもの)
- [ ] デプロイ手順が文書化されている  
- [ ] エラー監視・ログ収集の仕組みがある（任意: Sentry 等）  

## AI 利用

- [ ] AI に渡すデータに PII・機密が含まれない  
- [ ] AI 生成コードをマージ前に人がレビューしている  
  - 根拠: [実装規約.md §2](../実装規約.md#2-pr-粒度)
- [ ] AI に任せる範囲とレビュー責任者が明確

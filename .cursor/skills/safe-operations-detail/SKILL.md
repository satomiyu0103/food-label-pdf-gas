# 安全運用（詳細）

コア原則: `.cursor/rules/safe_operations_core.mdc`

## 例外分類

| 種別 | 応答 |
|---|---|
| 予期可能（重複・バリデーション） | スキップ + WARNING、バッチ継続 |
| 一時的（5xx・タイムアウト） | 指数バックオフリトライ（上限あり） |
| 永続的（認証・スキーマ不一致） | 当該単位失敗として記録 |
| 致命的（設定欠落・DB 不可） | 全体中止 FATAL + 即時通知 |

## 通知

- チャネルは `03_システム設計.md` または ADR で 1 本決める
- 本文: run_id / 環境 / 段階 / 対象 ID / 件数サマリ / 次のアクション
- 同一障害は 5〜15 分ウィンドウで重複抑制
- 通知失敗でも本処理は落とさない

## 処理量

- 実行前: 件数・API 消費見積と残枠比較
- 実行後: 実測をログ or 台帳に 1 行
- 枠不足時: 未処理を処理済みにしない
- 冪等性: 再実行で二重登録・二重課金にならない設計

## 新規バッチ最小セット

1. run_id + 段階ログ  2. 終了サマリ  3. API 上限方針  4. 障害通知チャネル  5. FR/NFR 追記
